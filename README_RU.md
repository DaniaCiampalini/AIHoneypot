## 🎯 Возможности

- **🎨 Интерактивная панель управления GUI**: Красивое настольное приложение JavaFX с мониторингом в реальном времени
  - Живая статистика угроз с автоматическим обновлением
  - Интерактивные графики (круговые и столбчатые диаграммы) для визуализации данных
  - Таблица последних угроз с подробной информацией
  - Анализ основных атакующих IP
  - Современная тема в стиле iOS с плавной анимацией
  
- **🚗 Симулятор трафика**: Генерирует реалистичный трафик honeypot для тестирования
  - Автоматическая генерация трафика каждые 5-30 секунд
  - Реалистичные шаблоны атак (SQL-инъекции, XSS, сканирование ботов)
  - Волны всплеска трафика для стресс-тестирования
  - Несколько типов клиентов (люди, боты, AI-агенты, сканеры)

- **🌱 Заполнение базы данных**: Предварительно заполняет базу данных историческими данными
  - 255+ начальных сеансов угроз
  - 7 дней исторических шаблонов
  - Реалистичные распределения атак по серьезности

- **🔍 Поведенческая дактилоскопия**: Анализирует шаблоны HTTP-запросов для различения людей и ботов

- **🧠 Многоуровневая классификация**: 
  - Обнаружение на основе правил для известных шаблонов
  - Isolation Forest для обнаружения аномалий
  - Ансамблевые методы, объединяющие несколько классификаторов

- **🐦 Обнаружение ботов X (Twitter)**: Специализированный модуль для обнаружения ботов и AI-агентов в социальных сетях
  - Анализ профиля (возраст, шаблоны имени пользователя, обнаружение био)
  - Сетевой анализ (соотношение подписчиков/подписок)
  - Временные шаблоны (частота публикаций)
  - Обнаружение текста, сгенерированного AI

- **🕸️ Канареечные ловушки**: Приманочные конечные точки, к которым не должен обращаться ни один законный пользователь
  - `/admin`, `/wp-admin`, `/.env`, `/backup`, `/.git` и другие
  - Автоматическая КРИТИЧЕСКАЯ серьезность при доступе

- **💾 Журналирование угроз в реальном времени**: Сохраняет сеансы угроз в базе данных H2/PostgreSQL

- **📊 Панель управления REST API**: Мониторит и анализирует обнаруженные угрозы через HTTP-конечные точки

- **🔄 Отслеживание сеансов**: Коррелирует несколько запросов из одного сеанса

- **🔒 Анализ безопасности**: Сканер безопасности веб-сайтов с 7-уровневым анализом
  - Проверка SSL/TLS
  - Проверка заголовков безопасности
  - Сканирование портов
  - Обнаружение уязвимостей

## 🏗️ Архитектура

Это многомодульный Maven-проект со следующей структурой:

```
AIHoneypot/
├── core/              # Доменные модели, интерфейсы, исключения
├── collector/         # Уровень сбора сигналов (фильтры сервлетов)
├── analyzer/          # Движок классификации угроз
├── dashboard/         # REST API и статистика
├── gui/               # Настольная панель управления JavaFX (НОВОЕ!)
├── x-detector/        # Модуль обнаружения ботов X (Twitter)
└── honeypot/          # Основное приложение Spring Boot + канареечные ловушки + симулятор трафика
```

### Технологический стек

- **Backend**: Spring Boot 3.2.2, Java 17
- **Frontend**: JavaFX 21 (Настольный GUI)
- **База данных**: H2 (in-memory) / PostgreSQL (production)
- **ML/AI**: Пользовательский isolation forest, классификация на основе правил
- **Графики**: JavaFX Charts API
- **Сборка**: Maven Multi-Module

## 🚀 Быстрый старт

### Предварительные требования

- Java 17+
- Maven 3.8+
- (Опционально) PostgreSQL для production

### Вариант 1: Полный запуск (Backend + GUI) - РЕКОМЕНДУЕТСЯ

Запустите все одной командой:

```bash
./start-complete.sh
```

Этот скрипт:
1. ✅ Проверяет, запущен ли backend
2. ✅ Собирает проект при необходимости
3. ✅ Запускает backend в фоновом режиме
4. ✅ Ожидает готовности backend
5. ✅ Запускает GUI панель управления

**Примечание**: Backend работает в фоновом режиме. Логи в `/tmp/aihoneypot-backend.log`

### Вариант 2: Ручной запуск

#### Запуск Backend

```bash
cd honeypot
mvn spring-boot:run
```

Приложение запустится на `http://localhost:8080`

#### Запуск GUI (в другом терминале)

```bash
cd gui
mvn javafx:run
```

Или используйте скрипт только для GUI:

```bash
./start-gui-only.sh
```

### Вариант 3: Только GUI (Backend уже запущен)

Если backend уже работает:

```bash
./start-gui-only.sh
```

Это проверит, что backend активен, и запустит GUI.

### Остановка Backend

```bash
# Найти и завершить процесс backend
pkill -f 'spring-boot:run'

# Или найти PID и завершить
lsof -ti:8080 | xargs kill -9
```

### Точки доступа

- **GUI панель управления**: Открывается автоматически при запуске `./start-complete.sh` или `./start-gui-only.sh`
- **Swagger API**: http://localhost:8080/swagger-ui.html
- **Консоль H2**: http://localhost:8080/h2-console
- **Документация API**: http://localhost:8080/api-docs
- **Проверка здоровья**: http://localhost:8080/actuator/health

## 📊 Основные классы домена

### Модуль Core (10 классов)

1. **ClientType** (enum) - HUMAN_BROWSER, AI_AGENT, BOT_SCRAPER, SEARCH_ENGINE, SECURITY_SCANNER, UNKNOWN
2. **Severity** (enum) - LOW, MEDIUM, HIGH, CRITICAL
3. **SignalType** (enum) - Типы сигналов для классификации
4. **RawRequestSignals** - Необработанные сигналы HTTP-запросов
5. **ClassificationResult** - Результат классификации
6. **BehaviorSignalExtractor** (интерфейс) - Извлечение поведенческих сигналов
7. **ThreatClassifier** (интерфейс) - Классификация угроз
8. **CoreException** - Базовые исключения
9. **SignalExtractionException** - Исключения извлечения сигналов
10. **ClassificationException** - Исключения классификации
